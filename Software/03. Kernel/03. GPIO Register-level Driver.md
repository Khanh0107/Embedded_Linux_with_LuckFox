# GPIO Register-level Driver

## 0. Introduction

### Why study GPIO Register-level Drivers?

* **GPIO is fundamental in Embedded Systems**
  Almost every embedded SoC (ARM, RISC-V, etc.) has GPIOs to control LEDs, buttons, and sensors.

* **Register-level = Direct register access**
  Instead of using the Linux GPIO subsystem (gpiolib), this driver directly **ioremap()s physical addresses** and manipulates the **GPIO controller registers** (`DR`, `DDR`, `IOC`).

* **Benefits**

  * Understand how **SoCs map GPIO into memory**.
  * Learn a **bare-metal-like** programming style within Linux kernel space.
  * A stepping stone toward writing **platform drivers** and integrating with the Device Tree.

---

## 1. Theoretical Foundations

### 1.1 GPIO in Linux: Memory-mapped I/O

* GPIO controllers are exposed as **physical memory-mapped registers**.
* Each **bank** manages around 32 pins.
* The kernel must **ioremap()** the physical base address to a **virtual address** before accessing.

Example:

```c
gpio_base[0] = ioremap(0xFF380000, GPIO_BASE_SIZE); // Map GPIO0
```

---

### 1.2 Important GPIO Registers

All below information is collected from the [RV1106 TRM](../../Docs/ReferenceManual/Rockchip_RV1106_TRM_V0.3-Part1-20220324.pdf).

#### Address Mapping:

| Module    | Physical Base Address | Size  |
| --------- | --------------------- | ----- |
| GPIO0     | FF380000              | 32 KB |
| GPIO0_IOC | FF388000              | 32 KB |
| GPIO1     | FF530000              | 32 KB |
| GPIO1_IOC | FF538000              | 32 KB |
| GPIO2     | FF540000              | 32 KB |
| GPIO2_IOC | FF548000              | 32 KB |
| GPIO3     | FF550000              | 32 KB |
| GPIO3_IOC | FF558000              | 32 KB |

#### IOMUX Select Registers (IOC):

<p align="center">
    <img src="../99. Images/Pasted image 20250921202450.png" width="700">
</p>

The **IOMUX register** controls the function of multiple pins at once.

- Each register typically configures **3 pins**.
- For each pin, **value `0x0` always selects GPIO mode**.
- To modify the function, the driver must also **set the corresponding 3-bit write-enable field in the upper 16 bits**. Without this, the lower 16-bit configuration will not take effect.

#### Port Data Direction Register (DDR):

<p align="center">
    <img src="../99. Images/Pasted image 20250921203017.png" width="700">
</p>
The **GPIO_SWPORT_DDR register** configures the input/output direction for the 16 pins of a GPIO bank.

- Controls direction for **16 pins** of the GPIO port.
- **Bit 0 = Input mode** (default), **Bit 1 = Output mode**.
- To modify direction, the driver must also **set the corresponding write-enable bit in bits 16-31**. Without this, the direction configuration will not take effect.

#### Port Data Register (DR):

<p align="center">
    <img src="../99. Images/Pasted image 20250921203428.png" width="700">
</p>

The **GPIO_SWPORT_DR** register configures the output data values for 16 pins of a GPIO bank.

- Controls output data for **16 pins** of the GPIO port.
- **Bit 0 = Low level** (default), **Bit 1 = High level** on the output signal.
- Only affects pins configured as **output** in the corresponding DDR register.
To modify output data, the driver must also s**et the corresponding write-enable bit in bits 16-31**. Without this, the data configuration will not take effect.

### 1.3 Data Flow: User → Kernel → GPIO Registers

```
echo "pin, value" > /dev/gpio_char
   │
   ▼
gpio_write()
   │
   ├── ioremap()   → map GPIO registers
   ├── writel()    → configure IOC (GPIO mode)
   ├── writel()    → set DDR (output)
   └── writel()    → write DR (pin value)
```

## 2. Core APIs

### 2.1 ioremap() / iounmap()

* `ioremap(phys_addr, size)` → maps physical memory into kernel virtual space.
* `iounmap(ptr)` → releases the mapping during cleanup.

### 2.2 readl() / writel()

* **readl(addr)** → read a 32-bit register.
* **writel(val, addr)** → write a 32-bit register.
* Ensure **MMIO barriers** for correct ordering.

### 2.3 struct file\_operations

```c
static struct file_operations fops = {
    .owner   = THIS_MODULE,
    .open    = gpio_open,
    .release = gpio_release,
    .read    = gpio_read,   // read DR
    .write   = gpio_write,  // set GPIO output
};
```

## 3. Implementation (Demo Code)

gpio_cdev.c

```c
#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/init.h>
#include <linux/fs.h>
#include <linux/cdev.h>
#include <linux/uaccess.h>
#include <asm/io.h>

#define DEVICE_NAME     "gpio_char"
#define GPIO_BASE_SIZE  0x8000U

static dev_t dev_num;
static struct cdev gpio_cdev;
static struct class *gpio_class;
typedef struct {
    unsigned int num;
    unsigned int bank;
    unsigned int port;
    unsigned int level;
} gpio_desc_t;

// Physical address of GPIO
static unsigned long gpio_phys[] = {
    0xFF380000, // GPIO0
    0xFF530000, // GPIO1
    0xFF540000, // GPIO2
    0xFF550000, // GPIO3
    0xFF560000, // GPIO4
};

static unsigned long gpio_ioc_phys[] = {
    0xFF388000, // GPIO0_IOC
    0xFF538000, // GPIO1_IOC
    0xFF548000, // GPIO2_IOC
    0xFF558000, // GPIO3_IOC
    0xFF568000, // GPIO4_IOC
};

static unsigned int iomux_sel_offset[] = {
    0x00, // GPIO0_A[0:3]
    0x04, // GPIO0_A[4:7]
    0xFF, // Not used
    0xFF, // Not used
    0xFF, // Not used
    0xFF, // Not used
    0xFF, // Not used
    0xFF, // Not used
    0x00, // GPIO1_A[0:3]
    0x04, // GPIO1_A[4:7]
    0x08, // GPIO1_B[0:3]
    0xFF, // Not used
    0x10, // GPIO1_C[0:3]
    0x14, // GPIO1_C[4:7]
    0x18, // GPIO1_D[0:3]
    0xFF, // Not used
    0x20, // GPIO2_A[0:3]
    0x24, // GPIO2_A[4:7]
    0x28, // GPIO2_B[0:3]
    0xFF, // Not used
    0xFF, // Not used
    0xFF, // Not used
    0xFF, // Not used
    0xFF, // Not used
    0x40, // GPIO3_A[0:3]
    0x44, // GPIO3_A[4:7]
    0x48, // GPIO3_B[0:3]
    0x4C, // GPIO3_B[4:7]
    0x50, // GPIO3_C[0:3]
    0x54, // GPIO3_C[4:7]
    0x58, // GPIO3_D[0:3]
    0xFF, // Not used
    0x00, // GPIO4_A[0:3]
    0x04, // GPIO4_A[4:7]
    0x08, // GPIO4_B[0:3]
    0xFF, // Not used
    0x10, // GPIO4_C[0:3]
};

// Virtual pointer after ioremap
static void __iomem *gpio_base[5];
static void __iomem *gpio_ioc[5];

static char kernel_buf[128] = "Hello from kernel!\n";

static int gpio_open(struct inode *inode, struct file *file) {
    pr_info("gpio_char: device opened\n");
    return 0;
}

static int gpio_release(struct inode *inode, struct file *file) {
    pr_info("gpio_char: device closed\n");
    return 0;
}

static ssize_t gpio_read(struct file *file, char __user *buf, size_t len, loff_t *offset) {
    u32 reg;
    reg = readl(gpio_base[3] + 0x0000); // Read DR_L of GPIO3
    snprintf(kernel_buf, sizeof(kernel_buf), "GPIO3 DR_L: 0x%X\n", reg);

    reg = readl(gpio_base[3] + 0x0004); // Read DR_H of GPIO3
    snprintf(kernel_buf + strlen(kernel_buf), sizeof(kernel_buf) - strlen(kernel_buf), "GPIO3 DR_H: 0x%X\n", reg);

    return simple_read_from_buffer(buf, len, offset, kernel_buf, strlen(kernel_buf));
}

static ssize_t gpio_write(struct file *file, const char __user *buf, size_t len, loff_t *offset) {
    gpio_desc_t gpio;
    u8 bit_order, mux_shift;
    u32 reg;

    size_t to_copy = min(len, sizeof(kernel_buf) - 1);
    if (copy_from_user(kernel_buf, buf, to_copy)) {
        return -EFAULT;
    }
    kernel_buf[to_copy] = '\0';


    if (sscanf(kernel_buf, "%u, %u", &gpio.num, &gpio.level) != 2) {
        pr_err("invalid format: echo \"pin_num, value\" > /dev/%s\n", DEVICE_NAME);
        return -EINVAL;
    }

    pr_info("gpio_char: Pin number %u, Level %u\n", gpio.num, gpio.level);

    // writel(0x0, gpio1_ioc + (pin * 4));
    gpio.bank = (gpio.num / 32);
    gpio.port = (gpio.num % 32);

    // Step 1: Set IOMUX to GPIO mode (0)
    mux_shift = (gpio.port % 4) * 4;

    reg = readl(gpio_ioc[gpio.bank] + iomux_sel_offset[gpio.num / 4]);
    reg = (reg & ~(0x7 << mux_shift)) | (0x7 << (mux_shift + 16));
    writel(reg, gpio_ioc[gpio.bank] + iomux_sel_offset[gpio.num / 4]);

    // Step 2: Set pin as output
    if (gpio.port < 16) {
        // DDR_L @ offset 0x0008
        reg = readl(gpio_base[gpio.bank] + 0x0008);
        reg |= (1 << gpio.port) | (1 << (gpio.port + 16));
        writel(reg, gpio_base[gpio.bank] + 0x0008);
    } else {
        // DDR_H @ offset 0x000C
        reg = readl(gpio_base[gpio.bank] + 0x000C);
        bit_order = gpio.port - 16;
        reg |= (1 << bit_order) | (1 << (bit_order + 16));
        writel(reg, gpio_base[gpio.bank] + 0x000C);
    }

    // Step 3: Output data for GPIO
    if (gpio.port < 16) {
        // DR_L @ offset 0x0000
        reg = readl(gpio_base[gpio.bank] + 0x0000);
        if (gpio.level == 1) {
            reg |= (gpio.level << gpio.port) | (1 << (gpio.port + 16));
        } else {
            reg = (reg & ~(1 << gpio.port)) | (1 << (gpio.port + 16));
        }
        
        writel(reg, gpio_base[gpio.bank] + 0x0000);
    } else {
        // DR_H @ offset 0x0004
        reg = readl(gpio_base[gpio.bank] + 0x0004);
        bit_order = gpio.port - 16;
        reg |= (gpio.level << bit_order) | (1 << (bit_order + 16));
        if (gpio.level == 1) {
            reg |= (gpio.level << bit_order) | (1 << (bit_order + 16));
        } else {
            reg = (reg & ~(1 << bit_order)) | (1 << (bit_order + 16));
        }
        writel(reg, gpio_base[gpio.bank] + 0x0004);
    }

    return to_copy;
}

static struct file_operations fops = {
    .owner   = THIS_MODULE,
    .open    = gpio_open,
    .release = gpio_release,
    .read    = gpio_read,
    .write   = gpio_write,
};

static int __init gpio_init(void) {
    unsigned int i = 0;
    if (alloc_chrdev_region(&dev_num, 0, 1, DEVICE_NAME) < 0) {
        pr_err("Failed to allocate character device region.");
        return -1;
    }
    cdev_init(&gpio_cdev, &fops);
    cdev_add(&gpio_cdev, dev_num, 1);

    // Create class and device file automatically
    gpio_class = class_create(THIS_MODULE, DEVICE_NAME);
    if (IS_ERR(gpio_class)) {
        pr_err("Failed to create class.");
        cdev_del(&gpio_cdev);
        unregister_chrdev_region(dev_num, 1);
        return PTR_ERR(gpio_class);
    }
    device_create(gpio_class, NULL, dev_num, NULL, DEVICE_NAME);

    // Map GPIO registers: physical address → virtual address
    for (i = 0; i < sizeof(gpio_phys)/sizeof(gpio_phys[0]); i++) {
        gpio_base[i] = ioremap(gpio_phys[i], GPIO_BASE_SIZE);
        gpio_ioc[i] = ioremap(gpio_ioc_phys[i], GPIO_BASE_SIZE);
        if (!gpio_base[i] || !gpio_ioc[i]) {
            pr_err("gpio_reg: ioremap failed for GPIO%d\n", i + 1);
            return -ENOMEM;
        }
    }

    pr_info("gpio_char: registered with major %d, minor %d\n",
            MAJOR(dev_num), MINOR(dev_num));
    return 0;
}

static void __exit gpio_exit(void) {
    unsigned int i = 0;
    // Unmap GPIO registers
    for (i = 0; i < sizeof(gpio_phys)/sizeof(gpio_phys[0]); i++) {
        if (gpio_base[i]) {
            iounmap(gpio_base[i]);
            iounmap(gpio_ioc[i]);
        }
    }

    device_destroy(gpio_class, dev_num); // Delete device file
    class_destroy(gpio_class);           // Delete class
    cdev_del(&gpio_cdev);
    unregister_chrdev_region(dev_num, 1);
    pr_info("gpio_char: unregistered\n");
}

module_init(gpio_init);
module_exit(gpio_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Tdieney");
MODULE_DESCRIPTION("GPIO Register-level Driver");
MODULE_VERSION("1.0");
```

## 4. Behavior Analysis

### Write Path

```
echo "123, 1" > /dev/gpio_char
   │
   ▼
gpio_write()
   ├─ parse pin=35 → bank=1, port=3
   ├─ configure IOC → GPIO mode
   ├─ set DDR_H → output
   └─ set DR_H → pin high
   
echo "123, 0" > /dev/gpio_char
   │
   ▼
gpio_write()
   ├─ parse pin=35 → bank=1, port=3
   ├─ configure IOC → GPIO mode
   ├─ set DDR_H → output
   └─ set DR_H → pin low
```

### Read Path

```
cat /dev/gpio_char
   │
   ▼
gpio_read()
   ├─ DR_L (GPIO3) → copy to buffer
   └─ DR_H (GPIO3) → append
```

## 5. Summary

* **ioremap()** maps physical GPIO registers into kernel virtual space.
* **readl/writel** provide MMIO access.
* Character driver exports `/dev/gpio_char` to user space.
* User can **echo / cat** to control or monitor GPIO pins.
