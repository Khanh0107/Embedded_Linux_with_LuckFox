# I2C - Read AHT30 sensor

This guide explains how to interface an AHT30 temperature/humidity sensor (I2C) with the Rockchip RV1106 board using U-Boot. It covers both the Device Tree configuration and the C code implementation, with detailed code comments and step-by-step instructions.

---

## 1. Device Tree Configuration (`rv1106-luckfox.dts`)

### I2C2 Controller
- The I2C2 controller is enabled for communication with I2C devices (like the AHT30 sensor):

```dts
&i2c2 {
    status = "okay";
    u-boot,dm-spl;
};
```

### pinctrl Section for I2C2 (Required)
To ensure I2C2 works correctly, you must configure the pinmux for the I2C2 pins in the `pinctrl` node. For example:

```dts
&pinctrl {
    pcfg_pull_none_smt: pcfg-pull-none-smt {
        u-boot,dm-spl;
    };

    i2c2 {
        i2c2m0_xfer: i2c2m0-xfer {
            u-boot,dm-spl;
        };
    };
};
```

The `i2c2m0-xfer` pinctrl group is defined in the `rv1106-pinctrl.dtsi` file as follows:

```dts
i2c2 {
    i2c2m0_xfer: i2c2m0-xfer {
        rockchip,pins =
            /* i2c2_scl_m0 */
            <1 RK_PA0 2 &pcfg_pull_none_smt>,
            /* i2c2_sda_m0 */
            <1 RK_PA1 2 &pcfg_pull_none_smt>;
    };
};
```

Then, reference this pinctrl in the I2C2 node:

```dts
&i2c2 {
    status = "okay";
    u-boot,dm-spl;
    pinctrl-names = "default";
    pinctrl-0 = <&i2c2m0_xfer>;
};
```

If you do not add this section, I2C2 may not work properly on some boards.

## 2. C Code Implementation (`evb_rv1106.c`)

### Reading the AHT30 Sensor via I2C

#### Key Steps:
1. **Get I2C Bus 2**: Use `uclass_get_device_by_seq` to get the I2C2 bus.
2. **Probe the AHT30**: Use `dm_i2c_probe` to detect the sensor at address `0x38`.
3. **Trigger Measurement**: Send the trigger command (`0xAC, 0x33, 0x00`).
4. **Read Data**: Read 6 bytes of sensor data.
5. **Parse Data**: Convert raw bytes to humidity and temperature values.

#### Code:
```c
#define AHT30_I2C_ADDR   0x38

static int read_aht30(void)
{
    struct udevice *bus, *dev;
    uchar cmd_trigger[3] = {0xAC, 0x33, 0x00};
    uchar buf[6];
    unsigned long raw_hum, raw_temp;
    int ret;

    // 1. Get I2C bus 2
    ret = uclass_get_device_by_seq(UCLASS_I2C, 2, &bus);
    if (ret) {
        printf("Cannot find I2C bus 2 (%d)\n", ret);
        return ret;
    }

    // 2. Probe the AHT30 at address 0x38
    ret = dm_i2c_probe(bus, AHT30_I2C_ADDR, 0, &dev);
    if (ret) {
        printf("Cannot probe AHT30 (%d)\n", ret);
        return ret;
    }

    // 3. Set Address offset (disable sub address)
    i2c_set_chip_offset_len(dev, 0);

    // 4. Trigger measurement
    ret = dm_i2c_write(dev, 0, cmd_trigger, sizeof(cmd_trigger));
    if (ret) return ret;
    mdelay(80); // Wait for measurement

    // 5. Read 6 bytes of data
    ret = dm_i2c_read(dev, 0, buf, 6);
    if (ret) return ret;

    // 6. Parse humidity and temperature
    raw_hum  = ((uint32_t)buf[1] << 12) | ((uint32_t)buf[2] << 4) | (buf[3] >> 4);
    raw_temp = ((uint32_t)(buf[3] & 0x0F) << 16) | ((uint32_t)buf[4] << 8) | buf[5];

    int humidity    = (raw_hum  * 1000) / 1048576;
    int temperature = ((raw_temp * 2000) / 1048576) - 500;

    printf("AHT30: %d.%d %%RH, %d.%d °C\n",
           humidity/10, humidity%10,
           temperature/10, temperature%10);

    return 0;
}
```
Invoke the function in your board initialization routine, typically in `rk_board_late_init()`:

```c
int rk_board_late_init(void)
{
    // ...existing code...
    read_aht30();
    // ...existing code...
}
```

## 3. How to Use

1. **Configure the Device Tree**: Ensure the I2C2 controller and (optionally) an initial pinctrl section are present as shown above.
2. **Build U-Boot**: Compile U-Boot with the updated device tree and C code.
Back to `../luckfox-pico/`  and run command

    ```sh
        ./build.sh clean uboot
        ./build.sh uboot
    ```
---

## 4. Additional Notes
- **I2C Address**: The AHT30 default address is `0x38`.
- **Debugging**: Use serial console to view the output of the print statements.
