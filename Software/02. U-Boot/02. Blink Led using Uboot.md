# Blink Led using Uboot

This guide explains how to configure and control GPIO LEDs on the Luckfox RV1106 board using U-Boot.

---

## 1. Hardware and Pin Mapping

- **LED1**: Connected to `GPIO3_PC6` (GPIO number **118**)
- **LED2**: Connected to `GPIO3_PD3` (GPIO number **123**)

## 2. Device Tree Configuration

Edit your device tree source (`rv1106-luckfox.dts`) to define the LEDs:

```dts
gpio-leds {
    compatible = "gpio-leds";
    status = "okay";
    u-boot,dm-spl;
    pinctrl-names = "default";
    pinctrl-0 = <&status_gpio_pins>;

    led1 {
        label = "status-led1";
        gpios = <&gpio3 RK_PC6 GPIO_ACTIVE_HIGH>;
        default-state = "off";
    };

    led2 {
        label = "status-led2";
        gpios = <&gpio3 RK_PD3 GPIO_ACTIVE_HIGH>;
        default-state = "off";
    };
};
```
- `compatible = "gpio-leds";`: Tells the system this node describes GPIO-controlled LEDs.
- `pinctrl-0 = <&status_gpio_pins>;`: Applies the pinmux group for the LED pins.
- Each `ledX` node describes one LED, its label, GPIO, and default state.

And ensure the pinctrl is set up:

```dts
&pinctrl {
    // This node defines a pin configuration with no pull-up or pull-down resistors.
    pcfg_pull_none: pcfg-pull-none {
        u-boot,dm-spl;
    };

    // This group assigns the correct pins and function for the LEDs.
    status_gpio_pins: statis-gpio-pins {
        rockchip,pins = <3 RK_PC6 RK_FUNC_GPIO &pcfg_pull_none
                         3 RK_PD3 RK_FUNC_GPIO &pcfg_pull_none>;
        u-boot,dm-spl;
    };
};
```
- `pcfg_pull_none`: Pin config with no pull resistor, referenced by the pin group.
- `status_gpio_pins`: Sets up both LED pins for GPIO function, using the above config.
---

## 3. U-Boot C Code for LED Control

In your board file (e.g., `evb_rv1106.c`), you can control the LEDs using the following function:

```c
// Blink both status LEDs 5 times during U-Boot initialization
static int blink_status_led_10x(void)
{
    int led1_gpio = 118; // GPIO3_PC6
    int led2_gpio = 123; // GPIO3_PD3
    int ret;

    printf("***** blink_status_led_10x *****\n");

    // Request control of LED1 GPIO. This reserves the pin for use.
    ret = gpio_request(led1_gpio, "status-led1");
    if (ret) {
        printf("Cannot request LED1 GPIO (%d)\n", ret);
        return ret;
    }
    // Request control of LED2 GPIO.
    ret = gpio_request(led2_gpio, "status-led2");
    if (ret) {
        printf("Cannot request LED2 GPIO (%d)\n", ret);
        gpio_free(led1_gpio); // Free LED1 if LED2 fails
        return ret;
    }

    // Set both GPIOs as outputs, initial value 0 (LEDs off)
    gpio_direction_output(led1_gpio, 0);
    gpio_direction_output(led2_gpio, 0);

    // Blink both LEDs 5 times: on for 500ms, off for 500ms
    for (int i = 0; i < 5; i++) {
        gpio_set_value(led1_gpio, 1); // Turn LED1 on
        gpio_set_value(led2_gpio, 1); // Turn LED2 on
        mdelay(500);                  // Wait 500ms
        gpio_set_value(led1_gpio, 0); // Turn LED1 off
        gpio_set_value(led2_gpio, 0); // Turn LED2 off
        mdelay(500);                  // Wait 500ms
    }

    // Release the GPIOs so other code can use them
    gpio_free(led1_gpio);
    gpio_free(led2_gpio);

    return 0;
}
```
Explanation of key steps:

- `gpio_request`: Reserves the GPIO for exclusive use.
- `gpio_direction_output`: Sets the pin as an output.
- `gpio_set_value`: Sets the output level (1 = on, 0 = off).
- `mdelay`: Waits for the specified number of milliseconds.
- `gpio_free`: Releases the GPIO when done.

Call this function in your board initialization, for example in `rk_board_late_init()`:

```c
int rk_board_late_init(void)
{
    blink_status_led_10x();
    return 0;
}
```
