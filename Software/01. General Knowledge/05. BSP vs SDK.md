# BSP vs SDK

## 1. BSP (Board Support Package)

**Definition:**
A BSP is a collection of **software components and configurations** required to make an operating system (usually Linux) run on a **specific hardware board**.
It is tightly coupled with the **hardware** (SoC + board).

**Main components:**

* Bootloader (e.g., U-Boot, SPL, vendor firmware)
* Linux kernel (with configuration and patches for the SoC/board)
* Device Tree (DTS/DTB) describing the board hardware
* Drivers for board-specific peripherals
* Patches, scripts, and hardware-specific settings

👉 **BSP = “software layer that makes the hardware usable with an OS.”**

---

## 2. SDK (Software Development Kit)

**Definition:**
An SDK is a **set of tools for developers** to build, compile, and debug applications that run on the target system.
It is more about **application development** than hardware bring-up.

**Main components:**

* Cross-compiler toolchain (gcc, g++, ld, etc.)
* Libraries (glibc, Qt, OpenCV, etc.) prebuilt for the target system
* Header files for development
* Debugging tools (gdb, strace, perf, etc.)
* Documentation, sample code

👉 **SDK = “tools for developers to build apps for that hardware/software platform.”**

---

## 3. Quick Comparison

| Aspect                | BSP                                        | SDK                                              |
| --------------------- | ------------------------------------------ | ------------------------------------------------ |
| **Purpose**           | Make the OS boot and run on the board      | Help developers build & debug apps for the board |
| **Main users**        | BSP engineers, system developers           | Application developers                           |
| **Components**        | Bootloader, kernel, DTB, drivers, patches  | Toolchain, sysroot, libraries, headers           |
| **Hardware relation** | Very hardware-specific                     | Less tied to hardware, focuses on software APIs  |
| **Example**           | U-Boot + Kernel + DTB for BeagleBone Black | Yocto SDK cross-compiler for Raspberry Pi        |

---

👉 In short:

* **BSP = to make the system run**
* **SDK = to develop applications for that system**
